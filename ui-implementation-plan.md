# План UI-реализации MCP Threat Analyzer

## Обзор
Документ описывает структуру UI и используемые компоненты shadcn для системы анализа угроз ФСТЭК.

---

## 1. Главная страница (Main Page)

### Верхняя панель (Header)
- **Separator** - разделитель между секциями
- **Button** - кнопка переключения темы
- **Button** - кнопка справки/описания методологии
- **Tooltip** - подсказки для кнопок

### Основная область
- **Card** - карточки навигации (4 карточки):
  - Карточка "Анализ угроз"
  - Карточка "Все угрозы"
  - Карточка "Меры защиты"
  - Карточка "Тактические задачи"
- **Badge** - индикаторы/метки на карточках (опционально)
- **Button** - кнопки перехода внутри карточек

### Информационный блок
- **Card** - карточка с информацией об источнике данных
- **Badge** - метки версии, даты актуальности

---

## 2. Экран анализа угроз (Threat Analysis Screen)

### Верхняя панель
- **Breadcrumb** - навигационная цепочка (Главная → Анализ угроз)
- **Button** - кнопка возврата на главную

### Основная рабочая область (двухколоночный layout)
- **Resizable** - разделение на левую и правую колонки с возможностью изменения размера

#### Левая колонка - Управление анализом
- **Card** - контейнер панели фильтрации
- **Accordion** или **Collapsible** - раскрывающиеся секции фильтров:
  - Категории угроз
  - Источники угроз
  - Тактические задачи
  - Меры защиты
- **Checkbox** или **Checkbox Group** - чекбоксы для выбора фильтров
- **Select** - выпадающие списки для фильтров (если нужен выбор одного значения)
- **Button** - кнопка "Сброс фильтров"
- **Separator** - разделитель между секциями
- **Badge** - отображение активных фильтров (с возможностью удаления)

#### Правая колонка - Результаты анализа
- **Card** - контейнер результатов
- **Input** - поле глобального поиска
- **Badge** - отображение количества найденных угроз
- **Table** - таблица угроз с колонками:
  - ID
  - Название
  - Тактическая задача
  - Нарушитель
  - Объект
  - Конфиденциальность/Целостность/Доступность (CIA)
  - Меры защиты
- **Pagination** - пагинация результатов
- **Empty** - состояние "Нет результатов"
- **Skeleton** - состояние загрузки

## 3. Страница списка угроз (Threats List Page)

- **Card** - контейнер панели управления

- **Select** - выбор сортировки

- **Empty** - состояние "Список пуст"
- **Skeleton** - состояние загрузки

## 9. Общие компоненты (используются на всех страницах)

### Навигация
- **Navigation Menu** - основное меню навигации (опционально, если нужна глобальная навигация)
- **Breadcrumb** - хлебные крошки для навигации

### Обратная связь
- **Toast** (Sonner) - уведомления о действиях пользователя
- **Alert** - важные сообщения/предупреждения
- **Alert Dialog** - диалоги подтверждения действий

### Вспомогательные компоненты
- **Tooltip** - подсказки при наведении
- **Skeleton** - индикаторы загрузки
- **Spinner** - спиннеры загрузки
- **Progress** - индикаторы прогресса (опционально)
- **Scroll Area** - области с прокруткой для длинного контента
- **Empty** - состояния пустых списков/результатов

### Формы и ввод данных
- **Input** - текстовые поля
- **Textarea** - многострочные поля ввода
- **Select** - выпадающие списки
- **Checkbox** - чекбоксы
- **Radio Group** - группы радиокнопок (если нужен выбор одного из вариантов)
- **Switch** - переключатели
- **Label** - метки для полей формы

### Модальные окна и выпадающие меню
- **Dialog** - модальные окна
- **Sheet** - боковые панели (опционально для фильтров на мобильных)
- **Drawer** - выдвижные панели (альтернатива Sheet)
- **Popover** - всплывающие окна
- **Dropdown Menu** - выпадающие меню
- **Context Menu** - контекстные меню (опционально)

### Дополнительные компоненты
- **Separator** - визуальные разделители
- **Badge** - метки и индикаторы
- **Kbd** - отображение горячих клавиш (опционально)

## 11. Структура данных и отображение

### Таблицы угроз
- **Table** с колонками:
  - ID (Badge)
  - Наименование
  - Тактическая задача (Badge)
  - Нарушитель
  - Объект
  - CIA метки (Badge для Конфиденциальность/Целостность/Доступность)
  - Меры защиты (список Badge)

### Карточки угроз
- **Card** с:
  - Заголовок (название угрозы)
  - Badge для ID
  - Описание (сокращенное)
  - Badge для тактической задачи
  - Button для перехода

### Детальная информация угрозы
- **Tabs** для организации:
  - Основная информация (Card)
  - Тактические задачи (Table)
  - Меры защиты (Table)
  - Связанные угрозы (Table)

## 12. Рекомендации по использованию компонентов

### Приоритет использования
1. **Card** - основной контейнер для контента
2. **Table** - для структурированных данных (списки угроз, мер, задач)
3. **Badge** - для меток, индикаторов, тегов
4. **Button** - для всех действий и навигации
5. **Input** - для поиска и фильтрации
6. **Select** - для выбора из списка
7. **Tabs** - для организации детальной информации
8. **Breadcrumb** - для навигации
9. **Separator** - для визуального разделения
10. **Empty** и **Skeleton** - для состояний загрузки и пустых результатов

### Компоненты для фильтрации
- **Accordion** или **Collapsible** - для группировки фильтров
- **Checkbox Group** - для множественного выбора
- **Select** - для одиночного выбора
- **Popover** - для компактного отображения фильтров

### Компоненты для навигации
- **Breadcrumb** - основная навигация
- **Button** - кнопки перехода
- **Card** - кликабельные карточки для навигации
